Xnoll Inventory - Documentation

This guide explains every module in the app one by one. Follow the exact steps here to onboard users and run daily operations.

Tip: Use keyboard shortcuts from the menu, such as Ctrl/Cmd + 1 to Ctrl/Cmd + 7 for quick navigation.

1. Getting Started

  1. Open the app and go to Settings first.
  2. Configure company details, currency, and stock behavior.
  3. Add warehouses before posting stock transactions.
  4. Create products and suppliers.
  5. Then start purchase orders and stock operations.

2. Menu Overview (Top Menu Bar)

  1. File: start new stock transaction, new purchase order, open settings, exit app.
  2. View: quickly open Dashboard, Products, Stock, Suppliers, Purchase Orders, Warehouses, Reports.
  3. Reports: direct shortcuts to Ledger, Reorder Alerts, Expiry Risk, Inventory Valuation.
  4. Help: open this documentation popup, online docs, website, support, and About dialog.

3. Dashboard Module

  1. Open Dashboard from sidebar or Ctrl/Cmd + 1.
  2. Review KPI cards: products, warehouses, suppliers, value, and alerts.
  3. Use dashboard as the daily health check before operations.

4. Products Module

  1. Open Products.
  2. Click add product and fill SKU, name, unit, category, price, and tax defaults.
  3. Use search/filter to find products quickly.
  4. Edit product details when pricing or metadata changes.
  5. Delete only unused products to preserve historical data consistency.

UnitMeasurement such as pcs, box, kg, ltr, service.

6. Purchase Orders Module

  1. Open Purchase Orders or use Ctrl/Cmd + Shift + N.
  2. Create PO with supplier, order date, expected date, and items.
  3. Set status (draft, ordered, partial, received, cancelled).
  4. Update PO as goods arrive and move to received state.
  5. Post stock transactions against received goods.

7. Stock Operations Module

  1. Open Stock Operations.
  2. Create stock entries for IN, OUT, or ADJUSTMENT.
  3. Select product, warehouse, quantity, and reference details.
  4. Use lot/batch/expiry fields if enabled in settings.
  5. Review movement history and running stock impact.

14. Custom Fields Module (Detailed)

Custom Fields lets you add dynamic fields to specific modules without code changes. Supported modules: customers, products, suppliers, warehouses.

How to Create a Custom Field

  1. Go to Settings > Custom Fields and click + Add Field.
  2. Enter Label (human name) and Name (key) (lowercase_underscore).
  3. Select target Module and Type (text, number, date, select).
  4. For select, add comma-separated options.
  5. Set behavior flags and save.

Custom Field Configuration Fields

Label *Visible field title in forms and table header.
Name (key) *Unique machine key. Must be lowercase letters, numbers, underscore.
ModuleWhere field appears (customers/products/suppliers/warehouses).
TypeText, Number, Date, Select.
OptionsRequired for select type (comma-separated list).
Default ValueInitial value used when no saved value exists.
RequiredField must be filled in module form.
Display in GridShow as extra column in list table.
Display in FilterReserved for filter surfaces where enabled.
SortableMarks field as sortable in data table context.
SearchableMarks field as searchable in module context.

How It Works at Runtime

  1. Module loads base records plus custom field definitions for that module.
  2. Form renders an Additional Information section dynamically.
  3. On save, system writes custom values per record ID.
  4. On edit, system loads and hydrates previously saved custom values.
  5. If display_in_grid is on, values appear in module table columns.

Important Rules

  1. Do not rename Name (key) after go-live unless migration is planned.
  2. Use clear label naming conventions across modules.
  3. Prefer select for standardized values and reporting consistency.
  4. Use required flag only for truly mandatory business data.

15. Recommended Daily Flow

  1. Check Dashboard for quick status and alerts.
  2. Review Reorder Alerts report.
  3. Create Purchase Orders for required items.
  4. Post Stock Operations for received/outgoing stock.
  5. Create Invoices and post payments.
  6. End day by checking Ledger and Valuation reports.
